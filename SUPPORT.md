# Support

## Getting help

- **Usage questions**: open a GitHub Discussion or Issue
- **Bug reports**: open a GitHub Issue with steps to reproduce
- **Feature requests**: open a GitHub Issue with use case details

## Before opening an issue

1. Read `README.md` and `docs/usage.md`
2. Check existing issues to avoid duplicates
3. Run `python -m pytest -q` to validate your local setup

## Security issues

For security-related reports, follow `SECURITY.md` and avoid public disclosure.
